Blast Royale : Game Release Checklist

Created by Miguel Tomas, last modified by Carlos on Jan 29, 2025

Client Checklist

  • PR merged release/* branch to master
  • PR merged release/* branch to develop
  • Android new version build to Google Play with STORE_BUILD & LIVE_SERVER symbols
  • iOS new version build to Apple TestFlight with STORE_BUILD & LIVE_SERVER symbols
  • iOS Setup Pass
  • Android Setup Pass
  • iOS Golden Pass
  • Android Golden Pass
  • iOS Negative Pass
  • Android Negative Pass

Backend Checklist

  • Ensure game builds, quantum builds and quantum server builds have no debug dlls
  • Announce to the community that the game will go into maintenance mode for a game update
  • Put the game in maintenance mode https://firstlightgames.atlassian.net/wiki/x/EgAMlQ
  • No players for 5min in any Quantum rooms in the current quantum cloud server and QA check if the live version is in maintenance
  • Backend deployed to LIVE_SERVER in Azure environment
  • Remove the staging redirect Environment Redirects

  • Go to front door resource click purge cache

  • Run any migration scripts on the Backend
  • Check for Unity Cloud builds for quantum dlls
  • Download client version from Apple & Google Play internal testing link
    https://testflight.apple.com/join/Lx4QhskZ
    https://play.google.com/apps/internaltest/4701681286424824904
  • Test with newly created account & change cosmetics in main menu
  • Test with existing account with Player Title Data from previous live version & change cosmetics in main menu
  • Check backend logs for when changing or equipping NFT-locked cosmetics
  • Play a ranked match between an iOS account against Android account
  • Check backend logs for the end of the match command
  • E-mail photon asking for metrics already
  • Bump game version in https://firstlightgames.atlassian.net/wiki/x/EgAMlQ
