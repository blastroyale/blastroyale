After we have an .aab file we need to upload it to google console, currently we can do in two ways:

  • Unity Cloud Builds: Does this automatically using fastlane supply, this is run by the Scripts/ucb_publish_google_play.sh configured in post completion scripts in the UCB build configuration.

    • The $PLAYSTORE_KEY is set via credentials in UCB. This is the json of a google cloud service account used to upload the .aab, you can find this file here in 1password.

  • Locally:

    • You can upload manually in google play console, inside the internal testing page, create a new release and upload the .aab file. https://play.google.com/console/u/0/developers/7073548425372989734/app/4974856990698981405/tracks/internal-testing

    • Run the script inside the repo Scripts/Dev/publish_google_play.sh this expects a BlastRoyale.aab inside the Unity Project folder and the service_account.json inside secrets. Also you gonna need Ruby, Bundler and fastlane to execute it. https://docs.fastlane.tools/getting-started/ios/setup/
