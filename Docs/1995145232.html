Setting up fresh NavMesh for a new map

Open your new scene and add 3D Object ‚Üí Terrain into Hierarchy

Rename the newly added GameObject to be ‚Äú<Mapname>Terrain‚Äù

In the Inspector ‚Üí Terrain Collider component you need to disable tree colliders as we don‚Äôt need them

Then let‚Äôs go into other terrain settings

As we are using Unity Terrain just for NavMesh we are interested only in 3 settings there

  • Detail Resolution Per Patch defines how big the ‚Äúcells‚Äù of the terrain are. To have enough flexibility with the nav mesh, set this value to 8

  • Terrain Width and Height should be roughly set to the size of the map. You can see the terrain in the scene window

When settings are defined - move the terrain in the scene window to align it with the map

Now you can save the project. You can see that there‚Äôs a new terrain data created, with a new file added to the root of the project. It‚Äôs called ‚ÄúNew Terrain‚Äù so rename it into ‚Äú<Mapname>Terrain‚Äù

Now create (if it doesn‚Äôt exist) the folder for design files of the map. We have a folder Design ‚Üí Mapworks. Create <Mapname> folder there. Then move your new terrain data inside that folder

Finally, drag the terrain gameobject from the hierarchy of the scene into Design ‚Üí Mapworks ‚Üí <Mapname> folder to create a prefab of the terrain

All setup is done now. Whenever you need to change navmesh you need to open the map scene, find terrain prefab, drag&drop it into the scene, make changes in the prefab, then create new nav mesh, and then delete that terrain prefab from the scene before saving the scene.

Changing and rebaking NavMesh

To begin with - open you <Mapname> scene. Then find correspondent terrain prefab in Design ‚Üí Mapworks ‚Üí <Mapname> folder, then drag&drop that terrain prefab into the hierarchy of the scene

To ‚Äúpaint‚Äù nav mesh using Unity Terrain we actually paint holes in the terrain. To do so you need to select Terrain object, select ‚ÄúPaint Terrain‚Äù tool and then ‚ÄúPaint Holes‚Äù in the dropdown, then choose a brush ‚ÄúNavMeshTerrainSquareBrush‚Äú

With the brush selected, you can start painting holes on top of obstacles and places where you don‚Äôt want NavMesh to be present. Orange outline will work as a guidance for you.

To erase holes you need to hold Shift button and ‚Äúpaint‚Äù where you want holes to be erased

Imagine there was an excessive hole in the ground and you removed it by ‚Äúerasing‚Äù holes

Now you are ready to bake. Disable all objects in the hierarchy except the NavMesh and Terrain. Select NavMesh gameobject. Make sure that in ‚ÄúUseGeometry‚Äù you have Physics Colliders and in Default Area you have Walkable. Then hit Bake

If you have gizoms active you will see how navmesh becomes the same as terrain

Old way of building navmesh

This is how old maps are working, except for Maze Mayhem

The navmesh will not build unless the mesh renderers themselves are enabled.

The navmesh should be built out of the collision layer of the map.

Process for building navmesh:

  • Search Coll_ in hierarchy

  • For all collision meshes, enable their mesh renderers

  • Build navmesh through unity

  • then build everything through quantum

  • Disable collision mesh renderers again

Make sure no other objects have any kind of navmesh baking inbuilt, or it will bake more complicated stuff and tank the performance.
