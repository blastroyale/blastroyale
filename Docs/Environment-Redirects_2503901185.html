Blast Royale : Environment Redirects

Created by Carlos, last modified on Oct 16, 2024

This is used to change the environment of certain builds, this is configured remotely using Azure Blob Storage. The main use case is in the release process when we have a store build pointing to production but want to redirect the new builds to staging before publishing them.

How to setup

  1. Go to â€˜blastroyalecdnâ€™ blob storage inside Azure and go to the blob container called â€˜buildsâ€™

  2. Duplicate the example file and name it following this rule platform_buildnumber.json, like Android_30080.json or IPhonePlayer_30079.json Remember Unity Cloud Build numbers start with 300

  3. Purge the cache inside blastroyalecdn Front Door resource Link.

    1. Click Purge Cache

    2. Select endpoint cdn, then all domains, check Purge all assets and click purge.

  4. Finished, test build

How to disable it?

  1. Delete the .json file from the blob container.

  2. Clean the cache, same as setting it up.

How can I know if redirect is working?

  • Check for logs containing Downloading redirect info from and Redirecting to.
