Setting up your Git Client and cloning the repo:

  1. Install your favorite Git Client.

  2. Make sure to install Git LFS (Large File System)

    • For Windows: https://git-lfs.github.com/

    • For Mac: brew install git-lfs && git lfs install (if home brew command is â€˜not foundâ€™ go to: https://brew.sh/ and install via terminal following the instructions.

  3. Clone the repository:

    • SSH: git clone -b develop git@bitbucket.org:first-light/blast-royale.git

Double-check if youâ€™re cloning from the develop branch!

Installing the Unity Editor and adding the Blast-Royale project to Unity Hub:

  1. When you begin installing Unity, you'll encounter an 'Add Module' step where you'll need to check and install the following modules:

    • Android Modules:

      • OpenJDK

      • Android SDK & NDK Tools

    • iOS Build Support

  2. On Unity Hub, open the 'Projects' tab and open the 'blast-royale' project that you cloned during the previous step, then select BlastRoyale directory

Setup the Photon Quantum - Netcode Framework:

You can skip this step if you are not a coder!!!

For Mac users:

  1. Download and Install the MonoFramework MDK

  2. Install OS environment variables:

    • For zsh users:

      • Open ~/.zprofile and paste the code below in, replacing <version> with the Unity Current Version. 

        export JAVA_HOME=/Applications/Unity/Hub/Editor/<version>/PlaybackEngines/AndroidPlayer/OpenJDK
export ANDROID_SDK_ROOT=/Applications/Unity/Hub/Editor/<version>/PlaybackEngines/AndroidPlayer/SDK
export ANDROID_NDK_ROOT=/Applications/Unity/Hub/Editor/<version>/PlaybackEngines/AndroidPlayer/NDK
grep -E "^setenv" /etc/launchd.conf | xargs -L 1 launchctl

    • For bash users:

      • Open ~/.bash_profile and paste the code below in, replacing <version> with the Unity Current Version. 

        export JAVA_HOME=/Applications/Unity/Hub/Editor/<version>/PlaybackEngines/AndroidPlayer/OpenJDK
export ANDROID_SDK_ROOT=/Applications/Unity/Hub/Editor/<version>/PlaybackEngines/AndroidPlayer/SDK
export ANDROID_NDK_ROOT=/Applications/Unity/Hub/Editor/<version>/PlaybackEngines/AndroidPlayer/NDK
grep -E "^setenv" /etc/launchd.conf | xargs -L 1 launchctl

    • Dock / Launcher / Finder:

      • As SUDO, open the /etc/launchd.conf (or create if it doesnâ€™t exist) and paste the code below in, replacing <version> with the Unity Current Version. 

        setenv JAVA_HOME /Applications/Unity/Hub/Editor/<version>/PlaybackEngines/AndroidPlayer/OpenJDK
setenv ANDROID_SDK_ROOT /Applications/Unity/Hub/Editor/<version>/PlaybackEngines/AndroidPlayer/SDK
setenv ANDROID_NDK_ROOT /Applications/Unity/Hub/Editor/<version>/PlaybackEngines/AndroidPlayer/NDK

You must REBOOT for changes to take effect!

For Windows users:

  • For Windows users, we will use the Visual Studio Installer (VS Installer) to install or modify the tools and frameworks required to run the project. If you already have Visual Studio installed, you probably already have it. Double-check by searching for 'Visual Studio Installer' in the Windows Search/Start Bar.

    • In case you donâ€™t find it, download the Visual Studio here and install it.

  1. Open Visual Studio Installer:

    • If you already have any Visual Studio Installation, click on 'Modify', otherwise proceed the installation.

  2. Switch to 'Individual components' tab.

    • You can access this tab during a new VS Installation process or modifying the installation later on.

  3. Under .NET select the following items:

    • .NET Framework 4.8.0 SDK

    • .NET Framework 4.8.0 Target Packing

  4. Click on 'Modify'

Regardless of the operating system you're using, you'll need to build the Quantum solution before opening the Unity project for the first time, and every time the Quantum project files are changed.

Please refer to the F.A.Q. - Quantum for further instructions how to do so.

Ignition sequence start! :

  1. Open the project you previously added to Unity Hub.

  2. Open the 'Boot' Unity Scene and hit Play. This is the main project scene.

Sometimes git can include permissions on the files, creating lots of unneeded files. Depending how is your OS, going to the root folder of the project and running the following command can help mitigate the issue:

git config core.fileMode false
