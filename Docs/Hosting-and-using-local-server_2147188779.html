Blast Royale : Hosting and using local server

Created by Gabriel, last modified by Carlos on Nov 28, 2024

Game Logic

Sometimes you want to develop something on the game while running a local server, mainly when you are changing Logic Commands inside the game or adding new items, so you need the game logic server locally.

We do it by sending the commands directly to the local server instead of sending the ‚ÄúExecuteCommand‚Äù to PlayFab API.

If you change any config or command remember to copy the game DLLs to the local server, you can do it by the topbar menu in Unity Editor FLG -> Backend -> Copy DLLs, remember to restart the server.

Requirements

Docker or .net core 6

How to Run Docker (Optional)

In your terminal of choice:

docker-compose build¬†and¬†docker-compose up

This should spin up an instance of the server and its required dependencies such as database.

How to Run on Rider/Visual Studio

Open the GameLogicServer and click Play

How to Connect

After running the local server, in your unity, find the First Light Games menu on the top bar.

FLG -> Local Flags -> Use Local Server

After clicking this menu for this session all logic requests will be sent to the local backend instead.

Debugging & Local App

When running directly from IDE you can add breakpoints normally.

Blast Hub

You can define to which blast hub your local server points in case you want to test some specific integration.

For local runs that can be done in the StandaloneServer Properties -> launchSettings.json
