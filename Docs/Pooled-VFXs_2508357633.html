Blast Royale : Pooled VFXs

Created by Carlos, last modified on Oct 23, 2024

Creating a new one

Expand

Add a new VfxId

  • Go to Assets/Src/FirstLight/Game/Ids/VfxId.cs and add a new value to the enum.

Create a prefab

  • Go to Assets/AddressableResources/VFX/Pooled and create a new prefab there, this will be spawned during the game with the MatchServices.VFXService.Spawn().

  • Add one of the following components

    • VfxMonoBehaviour - Simplest one, code is responsible for despawning it and returning to the pool with MatchServices.VFXService.Despawn() or the VfxMonoBehaviour.Despawn()

    • TimedVfxMonoBehaviour - automatically despawns after given time

    • ParticleVfxMonoBehaviour - automatically despawns based on ParticleSystem duration

  • Configure the component with the right values (VFXID, ParticleSystem etc.)

  • Make sure all VFX asset dependencies are in the same addressable bundle to avoid loading other bundles.

Add to the config

  • Add a new value in Assets/AddressableResources/VFX/PooledVFXConfigs.asset with the correct VfxId and the newly created addressable.

    • There is also InitialPoolConfig, this value is used to initialize a few instances during match loading.

Using the system

  • For spawning vfx you use the MatchServices.VFXService.Spawn method, it will return a reference of the VFX Component, you are responsible for despawning it to return it to the pool (except the ones with components that auto despawn the prefab (check creating a new one section)

  • After you got the reference you move the GameObject to the place you want or attach it to another transform.

  • All VFXs are destroyed on the match end
