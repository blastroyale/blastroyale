Blast Royale : Event Notifications Config

Created by Carlos, last modified on Nov 25, 2024

Click here if you don't know how to change values 

{
    "ScheduleHoursBefore": 48,
    "ShowWhenAppIsOpen": false,
    "Notifications": [{
        "TimeBefore": 0,
        "Messages": [{
            "Title": "Event %event_title% has just started. Let's GO!",
            "Description": "%event_long_description%"
        }, {
            "Title": "Event at %starts_at% has started. Let's GO!",
            "Description": "%event_long_description%"
        }]
    }, {
        "TimeBefore": 5,
        "Messages": [{
            "Title": "Event %event_title% starts in 5 minutes!",
            "Description": "%event_description%"
        }, {
            "Title": "Don't miss the event at %starts_at%",
            "Description": "%event_long_description%"
        }]
    }]
}

Description of the fields:

  • ScheduleHoursBefore: This is used to schedule notifications, everytime the player opens the game it searches for events in the next X hours to schedule notifications, a higher value means players can receive outdated info about events.

  • ShowWhenAppIsOpen : Doesnâ€™t show the notifications if the player is playing the game

  • TimeBefore: Minutes before the event to send the notification

  • Messages: It can have multiple entries with Title and Description, it picks a random one when sending to the player, you can use the following placeholders:

    • %event_title% - Event title from Event configs

    • %event_description% - Event description from Event configs

    • %event_long_description% - Event long description from Event configs

    • %starts_at% - Time when the event starts in the given format 5:45 PM
