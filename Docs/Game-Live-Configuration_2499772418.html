Blast Royale : Game Live Configuration

Created by Carlos, last modified on Oct 15, 2024

Summary

  • Go to Unity Cloud

  • Go to Remote Configuration

  • Change the JSON values you want.

  • Bump the ServerConfigurationVersion number.

  • Publish changes.

How does it work?

Client-side

We store our live operable game configurations inside Unity Cloud Remote Configs, then players fetch those configs every time they open the game, and also fetch updates before joining matchmaking. So itâ€™s important to bump the version number because the client skips updates if he already downloaded that version number.

We can also control how frequently we want to fetch remote configs, because itâ€™s an external request every time a player clicks to play, in my tests so far this is not a problem, but you can configure it in GeneralConfig field ConfigCacheInSeconds.

Server-side

When sending commands to the server, players also send the configuration version they are using so the server can update if needed, or if the server is ahead of the player forcing them to update configs.

Currently server only updates configs when a player sends a higher number than the current cached one, this has one issue, if no one sends a command after we update a config, and a match ends the server will calculate rewards based on older configs. This is something that needs to be fixed.
