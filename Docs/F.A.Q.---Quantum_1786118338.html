Blast Royale : F.A.Q. - Quantum

Created by Miguel Tomas, last modified by Carlos on Nov 08, 2022

The Project Phoenix unity project relies upon some assemblies and generated code that are built by the separate Quantum Solution found at quantum_code/ in the repository. These generated files are not checked into source control to minimise the conflicts they might cause when merging. Instead, you will need to generate these files on your local machine when you work with the Unity project.

When does the Quantum Project need to be built?

  1. If the game repository has been freshly checked out you will need to build these files for the first time.

  2. If you are working directly on the Quantum code files you will need to rebuild every time you want to use your changes in the Unity editor.

  3. Every time you perform a git operation that results in changes to the Quantum project files. For example switching branches, or merging in changes from another branch.

What do I need in order to build the Quantum Project in an IDE?

To build the Quantum project you will need to have one of the following on your local machine:

  • Visual Studio Community (win) - https://visualstudio.microsoft.com/vs/

  • Visual Studio for Mac (mac, obviously) - https://visualstudio.microsoft.com/vs/mac/

  • Rider - https://www.jetbrains.com/rider/

  • For VSCode, use the command line build step below

If you are a Windows user using Visual Studio you will also need to install the dotnet 4.6.2 Targeting Pack and 4.6.2 SDK afterwards. You may have already done this as part of Project Setup

How do I build the Quantum Project?

To build the project in the IDE:

  1. Open the Quantum Solution file Quantum/quantum_code/quantum.code.sln in Visual Studio or Rider.

  2. Build the solution from the menus

    1. In Visual Studio Build => Build All

    2. In Rider Build => Build Solution

Thatâ€™s it. For reference building the Quantum Solution should take < 5 seconds.

To build the project on the command line:

Mac:

msbuild \
./Quantum/quantum_code/quantum_code.sln \
-restore \
-p:Configuration=Release \
-p:RestorePackagesConfig=true

Windows:

msbuild ^
./Quantum/quantum_code/quantum_code.sln ^
-restore ^
-p:Configuration=Release ^
-p:RestorePackagesConfig=true
