Blast Royale : How to Community Builds

Created by Carlos, last modified on Aug 24, 2023

Build

  • Check if you bumped the version of the project since the last community build!

  • Trigger pipeline with the following parameters
    - Select the branch you want the source code from
    - Build Type: STORE_BUILD
    - Target Server Environment: LIVE_TESTNET_SERVER
    - Check [CD] Deploy Game Logic

  • Wait for pipeline to finish

Pre-Release

  • Go to a MacOS machine (can be the remote mac mini),

  • Download the .ipa file from the #builds channel on slack.

  • Open Transporter app and log into your apple developer account.

  • Click on the + icon and select the downloaded .ipa file

  • Wait for it to process and click ‚ÄúDelivery‚Äù

  • Go to https://appstoreconnect.apple.com/apps/1621071488/testflight/ios and wait for apple to process the version.

  • Add the ‚ÄúTemp Group‚Äù to the build, so we can trigger the review process without automatic notifying the community when it is done.

  • Wait for review to proceed to release

Release

  • Go to the Live Test Net playfab, go to Content ‚Üí Title Data ‚Üí Edit

  • Update the version to be the same as the build. This will block old versions of playing the game.

  • Go to azure devops and approve the deployment of the game logic service, it will be a pending deployment with the same build number of the game.

  • Add the user groups to TestFlight and Firebase build

  • Remove access to the old community build

  • Notify Renny that the deployment is ready

  • We are done
