Blast Royale : Commit message / branch naming convention

Created by Miha Krajnc, last modified by Miguel Tomas on Jul 14, 2022

Commit message:

  1. Should be written in imperative tense, I.E. When this commit is applied it will *commit message*.

  2. Capitalise the first word and do not end in punctuation.

  3. If the commit was made as a result of a JIRA task, the task should be at the start of the message, followed by a colon and a space.

  4. If you want to add more info to the message, separate it from the title by 1 empty line. It's ok to use bullet points (*).

Examples

  • Implement new bot AI system

  • BR-60: Fix skydiving speed issue

  • BR-210: Tweak bullet mechanich
    * Increased bullet speed.
    * Reduced bullet penetration.

Branches:

  • We should use the Git Flow strategy with some modifications:

  • Branches should be named all lowercase with underscores.

    • feature/ - new features or improvements

    • bugfix/ - bug fixes like during the QA

    • temp/ - experiments and potentially delete branches

    • hotfix/ - patch release branches

    • release/ - release branches during QA week

  • All features / tweaks / improvements should be in a branch starting with feature/

  • Branch names should be all lowercase with _ or - separators.

  • Bugs should be fixed in branches starting with bug/

  • Temporary branches should start with temp/ (and potentially get automatically deleted after a while)

  • [OPTIONAL / TO DISCUSS] Branches are merged to develop with a merge commit, and rebased before the merge (semi-linear history).
