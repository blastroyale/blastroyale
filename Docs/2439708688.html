Blast Royale : Content Pipeline / Import Skins

Created by Gabriel, last modified by Carlos on Oct 14, 2024

We want to produce high quality content as fast as possible, reason why we created an content pipeline.
This is currently how our pipeline flow works.

We need one operator for this flow, which is a person able to open unity and click few buttons, it does not require to be an expert but more knowledge is better when issues arise.

We separate content into types, for instance skins, flags, avatars etc and we keep them separate in Unity Asset Manager.

The process being with Artists putting the assets into the Asset Manager.

Before we start, let‚Äôs ensure our local asset manager is configured to properly create folders ensuring the checkbox is marked:

Then, to open the asset manager in Unity you need to click here:

This will open the asset manager. You can see on the left side the collections we have, already divided as the artists already place things in the correct place.

Assets that are not imported will not have a green checkmark on them.

When selecting the asset you can visualize some of its details and click import to begin the importing process.

Now the skin is imported, and ready to be tested in-game.

Simple as that !!!

Nerd Note

The import flow is done via Asset Importers, for example this is our character importer.
The importer will pick the assets from AssetManager and:

  • Move them to the correct collections folder

  • Create a GameId for them

  • Add them to the relevant configs, for instance our Skins configs in this case

Edge Case: Game Ids did not work

Sometimes quantum might not re-build automatically or game-ids might not update automatically.

You might need to re-generate game-ids

Then re-build quantum (you will need to know or ask tech to do it for you)

And finally fix the config that likely added the new entry with ‚ÄúRandom‚Äù id as the game id was not properly generated
