Blast Royale : "Painting" ground & bushes using GSpawn tool

Created by Nik Denissov, last modified on Oct 27, 2023

Overview

We use GSpawn tool to paint ground out of tiles. It‚Äôs just a visual, no colliders. We will create a single collider for the ground and in places like rivers we will setup invisible walls so players can‚Äôt walk into water.

All ground tiles are 2x2 size - it‚Äôs a small enough size to allow us to be flexible with the structure, but also big enough to have room to create variations of tiles.

We will also use GSpawn to paint bushes as the tool has automatic snapping and alignment which is convenient for placing dozens of similar objects.

We will NOT use GSpawn for:

  • navmesh (we use Unity Terrain for it)

  • collisions (this is done manually and/or per prefab)

  • placement of buildings and obstacles (this is done with drag&drop manually)

GSpawn tool overview and setup

If it‚Äôs a first time use of GSpawn then make sure that Grid settings are correctly set for ground tiles with height 2.

Go to Grid settings

And make sure that local Y offset is set to -1

This offset will create all ground tiles lower, so the 2x2x2 box will have its top surface at scene Y = 0

This offset is committed into the repository so you don‚Äôt need to set it up every time.

Now let‚Äôs look at how to setup prefabs.

Open the scene you want to work with.

Then initialize GSpawn

You should see GSpawn gameobject added into Hierarchy

Now open Prefab Library Manager

As with grid settings, this prefab library should already contain items. If it doesn‚Äôt then find in the project the following folder: Art ‚Üí Prefabs ‚Üí Environment ‚Üí Tiles. Then drag&drop this Tiles folder into Prefab Lib Db window

Note: you can‚Äôt drag individual prefabs into GSpawn library, you have to drag folders.

You should get something like this

To see how prefabs actually look let‚Äôs open Prefab Manager window

The last part is to setup Random Prefabs. Again, this is something that will be already setup so only necessary to do it once (and when we add more random prefabs).

Open the Random Prefabs window

There you can press dropdown to see various profiles or create a new one. We will need one profile per each type of ground - one for grass, one for sand, one for tarmac etc.

Let‚Äôs say we created a new profile RandomGrass, but it‚Äôs empty for now

To fill the profile you simply drag the necessary prefabs from Prefab Manager into Random Prefabs

So now we have a RandomGrass profile with 4 variations of Grass tile. With the way GSpawn works we will need to have separate prefabs, we can‚Äôt swap materials.

Why do we need 4 prefabs for the same grass? It‚Äôs to provide a subtle variety.

Imagine we have 2 sand tiles like these:

They are generally the same, but the left one has some sort of bigger bump in the bottom left, while the right one has a much smaller bump middle bottom. Spawning these prefabs randomly we get a coherent yet not ‚Äúcloned‚Äù feel for the ground.

How do the prefabs get randomized? To do this we specify a chance for each prefab in Random Prefabs window. Tap a prefab and then set the probability - range from 0 to 1 - this is sort of ‚Äúweight‚Äù, not % chance, as it is set per prefab individually.

In addition to random prefabs, we will make our own tool to apply random rotation (with a 90-angle step) to ground tiles.

It means that we will have 4 tiles * 4 rotations = 16 subtle variations

Painting Ground

To place many tiles at once out of Random Prefab you need to select GSpawn gameobject in hierarchy, then choose Object Spawn mode and then Box mode

In Prefab Pick Mode choose Random

Then in Random prefab profile choose the desired profile

Then in Prefab Manager double click on any prefab that is part of the Random Profile you chose. In this case I‚Äôve double clicked Grass1 prefab. Then move your mouse to the scene (it‚Äôs better to have camera oriented top down). You should see the cursor of your mouse turned into Grass prefab

Now click where you want grass to start, then move mouse to where you want grass to end

Click again and you will see how this area is automatically filled with random grass prefabs (according to probabilities set)

If you need to paint just a single tile then choose Modular Snap tool in GSpawn

Then double click the desired prefab in Prefab Manager and then tap in the scene where you want to place this prefab.

All prefabs are being spawned at the top level of hierarchy, so to clean things up we can move those prefabs into Ground folder

The last step is to add 90-angle step randomization for those ground prefabs. It will be done with our own tool.

Deleting ground

To delete ground you can simply select prefabs in the scene and delete them, nothing fancy.

Mesh combine

Having so many prefabs for the ground is not cool, so there‚Äôs Mesh Combine tool provided in GSpawn.

In the tool window you setup source parent, destination parent, a few settings and press Combine Children

As a result you will have neatly made meshes

Source prefabs are not needed in the map scene so they can be turned into a big prefab and moved into MapWorks ‚Üí <MapName> folder in case we need to make adjustments later.

GSpawn gameobject is also not needed in the final scene and can be deleted after use.

Painting Bushes

To paint bushes you need to choose Props tool in GSpawn (icon with a tree)

Then double click Bush prefab in prefab manager and then drag in the scene to paint bushes

You can adjust Drag Spawn ‚Üí Min Drag Distance setting to make spawning more dense or sparse

You can also change randomized rotation rules. However, random 180 step rotation seems the best as our bushes aren‚Äôt squares

When finished painting - make sure to organize bushes into parents so separate blocks of bushes sit inside different parents
