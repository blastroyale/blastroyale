Blast Royale : Building Locally

Created by Carlos, last modified on Nov 28, 2024

Build and Run

You can build locally using the Build & Run provided by unity, but you have to remember to build addressables before it manually. If you donâ€™t change the player settings it will generate a Debug build targeting DEVELOPMENT environment.

Helpers

You can use helpers inside the Top Menu FLG/Build/Local this has some options to help building in one click, every option automatically builds addressables as well.

There are some options with pre configured setups

  • Develop â†’ Debug, Targeting Develop Server

  • Develop - Faster Build â†’ Same as Develop changing some IL2CPP generation settings to make the building process faster.

  • Develop - Clean Build â†’ Same as Develop Donâ€™t use addressables and game build cache

  • Staging - Store â†’ Debug Disabled, Targeting staging environment, Generates .aab file for google play. This should be identical on how unity cloud build production files.

The output binary will be located at the Unity Project repository with the following name: BlastRoyale.apk or BlastRoyale.aab or BlastRoyale.ipa depending on the platform you are targeting.

Extra Tips

  • Check the Scripts/Dev directory, there are some useful scripts to help test on device:

    • adb-game.sh - Reopen the game

    • adb-link.sh - Opens a link in the device

    • google_upload_internal.sh - Upload the previousBlastRoyale/BlastRoyale.aab to google play and returns a link to install on devices (adb-link is very helpfull with this). It uses this feature https://play.google.com/console/about/internalappsharing/

    • adb-uninstall-reboot.sh - Uninstall the game and reboot the device, also takes a link as parameter to automatically open when the device boots up. This is useful for reinstalling an STORE build, because google play caches the version it provides, some buill shit.
