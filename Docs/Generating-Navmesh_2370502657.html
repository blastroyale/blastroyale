Blast Royale : Generating Navmesh

Created by Carlos, last modified on Sep 06, 2024

This process uses QuantumStaticColliders as the source of the NavMesh, and requires the map to have objects/colliders organized by Floor and Obstacles, you can have multiple objects per group.

How to add this method to a map?

  1. Delete everything NavMesh related from the current scene, it is better to do it from scratch.

  2. Add the prefab Assets/Design/Mapworks/Navmesh Baking.prefab as a child of QuantumMap

  3. Configure the Obstacle inside the source list of colliders, like the screenshot below:

Ignore and remove the floor group in the screenshots, this is not used anymore, the floor collider is generated based on the quantum map size.

  2. Bake the navmesh clicking on ‚ÄúBake Navmesh‚Äù Pink button, then you will see some disconnected islands in the map like the image below. It is ok we are going to fix in the next step.

  3. Open the Cleaner children in the NavMesh Baking prefab, and then Ctrl + Click a point where players can walk in the navmesh, this point will be used to clip all the disconnected islands.

  4. Then go back to the prefab root, and click bake again, and the unity part is done.

  5. Now lets import into quantum, just go to Quantum Map, configure the Bake all mode the same way it is in in the screen shot, then click Bake All.

    9. Save the scene and we are done
